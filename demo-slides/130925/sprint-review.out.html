Tips

  • Each team has his own xsl!
    • Remaining consistent decks over different teams requires either:
      • one central xsl & templates
      • sync changes between team xsl's
  • Use an assisted editor like Eclipse
    • realtime validation through xsd
    • code completion
    • syntax highlighting
  • Run xsl tranformation for xsl editing
    • easy launch configuration xsd transformations (import into eclipse)
    • Feedback xsl transform. in Console View
  • Refresh-in-browser review.xml when editing the deck
    • Open de review.xml directly in browser
      • firefox: go to about:config
        • disable (set 'false') security.fileuri.strict_origin_policy flag
      • chrome:
        • start chrome with "chromium-browser --disable-web-security"
